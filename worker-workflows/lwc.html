Hard rules

Rule Why
Never deploy to non-spencerdev orgs Prod is read-only.
Never push or merge Feature branch only. User reviews diff.
Never git add -A / git add . Stage by explicit path.
Never modify .pm/, Apex, flows, or object metadata Wrong-lane work → BLOCKER → orchestrator re-routes.
Never skip pre-commit hooks Husky + lint-staged catch real issues.
Never override .slds-* classes Playbook LWC hard-stop — use the component's own scoped classes.
Never use lightning-button for Cancel / Confirm / Save / Approve Project rule — use native buttons with grey-hover / green-positive pattern.
Never stall waiting for input Ambiguity → BLOCKER → return.

What this worker can / cannot do

Can

  • Create new LWC / Aura bundles from the 13 starter templates
  • Edit existing components, add Jest tests, run lint + prettier + jest locally
  • Fetch SLDS 1 docs via gh api and SLDS 2 design context via Figma MCP
  • Deploy the whole component directory targeted to spencerdev
  • Apply project deltas: color tokens, native-button pattern, WorkatoService outbound, cacheable discipline

Cannot

  • Edit Apex / flows / object metadata / permission sets
  • Deploy single files — always the whole component dir
  • Override .slds-* globally — playbook hard-stop
  • Use lightning-button for Cancel / Confirm / Save / Approve
  • Skip pre-commit hooks
