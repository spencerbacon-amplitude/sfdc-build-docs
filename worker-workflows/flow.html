Build in Flow Builder, not raw XML write

  • force-app/main/default/flows/<FlowName>.flow-meta.xml

Hand-editing flow XML is allowed for SMALL textual changes only (description, label, version notes). Structural changes — adding elements, reordering connectors, modifying decisions — go through Flow Builder. If the task implies hand-editing XML to add a new element, BLOCKER + ask for confirmation.

Deploy as Draft shell gate

  • sf project deploy start --source-dir force-app/main/default/flows/<FlowName>.flow-meta.xml

Bulk-test before requesting activation shell gate

  • For record-triggered or batchable flows: validate against 251+ records
  • anonymous Apex script inserting/updating a batch · inspect results
  • document the bulk-test in the DONE report

Single-record happy path is not enough — flows hit limits at scale.

Parse deploy result shell

Common gotchas: referenced field/object not yet deployed (sequence the metadata worker first); storeOutputAutomatically with missing field definitions on the variable; required input variable missing on a subflow call; inactive referenced object/RecordType.

  • every DML has a fault path
  • transaction boundaries on screen flows
  • variable prefixes used: var_ / col_ / rec_ / inp_ / out_
  • Collection Filter preferred over Loop+Decision
  • Transform over Loop where applicable

Hard rules

Rule Why
Never deploy to non-spencerdev orgs Prod is read-only.
Never auto-activate a flow Deploy as Draft; activation requires explicit user approval.
Never push or merge Feature branch only.
Never git add -A / git add . Stage only the specific flow file(s) you touched.
Never modify .pm/, Apex, LWC, or object/permission metadata Wrong-lane → BLOCKER.
Never hand-edit complex XML structural changes Use Flow Builder for structure. Textual-only edits (description, label) are fine.
Never stall waiting for input Ambiguity → BLOCKER → return.

What this worker can / cannot do

Can

  • Author record-triggered, screen, autolaunched, scheduled, platform-event flows
  • Edit existing flow descriptions, labels, version notes by hand
  • Deploy as Draft to spencerdev
  • Run bulk-tests via anonymous Apex against 251+ records
  • Apply TriggerSettings__c bypass discipline and three-tier error handling

Cannot

  • Auto-activate flows — Draft only
  • Hand-author structural XML changes (Builder only)
  • Edit Apex / LWC / objects / fields / permission sets
  • Reference fields that aren't yet deployed
  • Skip the bulk-test / anti-pattern checklist
